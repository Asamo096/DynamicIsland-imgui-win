# DynamicIsland - 代码审查与修复规格文档

## 概述
- **摘要**：审查项目中全部代码，查找逻辑、语法等问题并进行优化修复，解决灵动岛鼠标截获问题。
- **问题分析**：当前实现中，鼠标过滤逻辑存在根本性缺陷 - `g_imguiWindowRect` 在消息处理时可能尚未初始化，导致所有鼠标消息被错误地透传给下层窗口。

## 发现的问题

### 问题 1：鼠标过滤逻辑在启动时错误工作
- **位置**：`IsPointInImguiWindow` 函数
- **问题**：`g_imguiWindowRect` 初始化为 `{0,0,0,0}`，导致函数在 ImGui 渲染前始终返回 `false`
- **影响**：启动时所有鼠标消息被 `DefWindowProc` 处理，ImGui 完全收不到鼠标事件

### 问题 2：鼠标消息过滤时机错误
- **位置**：`WndProc` 函数
- **问题**：鼠标消息在传递给 ImGui 之前就被过滤掉了
- **影响**：即使 `g_imguiWindowRect` 已正确设置，ImGui 也无法接收鼠标事件

### 问题 3：窗口全屏但内容只在一小区域
- **问题**：窗口是全屏的，但 ImGui 只在中央小区域绘制
- **影响**：用户点击非 Imgui 绘制区域时，期望事件传递给下层窗口，但当前实现让整个窗口都"不可交互"

## 解决方案

### 方案：完全移除鼠标过滤逻辑，依赖窗口层级和 WS_EX_TRANSPARENT

**核心思路**：不再尝试在运行时过滤鼠标消息，而是通过正确的窗口样式和层级管理来实现期望的行为。

1. 保持全屏窗口
2. 移除 `IsPointInImguiWindow` 和相关鼠标过滤代码
3. 保持 `WM_NCHITTEST` 返回 `HTTRANSPARENT` 的逻辑（这已经能处理非客户区的穿透）
4. 对于客户区，让 ImGui 自己处理（ImGui 会根据 `WantCaptureMouse` 标志决定是否捕获鼠标）

## 修改范围
- 删除 `g_imguiWindowRect` 相关逻辑（如果不需要）
- 删除 `IsPointInImguiWindow` 函数
- 删除 `WndProc` 中的鼠标消息过滤代码
- 保留 `WM_NCHITTEST` 的拖拽区域逻辑

## 验收标准
- AC-1：鼠标在灵动岛 Imgui 显示区域内时，ImGui 能正常捕获并处理鼠标事件
- AC-2：鼠标在灵动岛非显示区域（空白透明区域）时，鼠标事件能传递给下层窗口
- AC-3：文件拖放功能正常
- AC-4：快捷键功能正常
