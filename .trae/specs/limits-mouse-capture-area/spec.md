# DynamicIsland - 限制鼠标捕获范围至Imgui显示区域规格文档

## 概述
- **摘要**：限制 DynamicIsland 窗口的鼠标操作范围仅在 Imgui 显示区域内，解决鼠标被灵动岛窗口全部截获的 Bug。
- **目的**：确保鼠标只有在 Imgui 渲染的实际显示区域内才被捕获，非显示区域应透传给下层窗口。
- **目标用户**：使用 DynamicIsland 的 Windows 用户。

## 背景与上下文
- DynamicIsland 是一个 Windows 系统监控工具，模仿苹果的灵动岛设计
- 当前问题：鼠标事件被整个窗口区域捕获，包括非 Imgui 显示区域，导致用户无法操作窗口下方的其他应用
- 根本原因：窗口的鼠标捕获范围没有限制在 Imgui 渲染区域内
- 解决方案：通过区域检测，限制鼠标操作只在 Imgui 显示区域内响应

## 目标
- 鼠标事件仅在 Imgui 显示区域内被捕获
- 非 Imgui 显示区域（即窗口的透明/空白区域）不捕获鼠标事件，鼠标事件透传到下层窗口
- 保持 Imgui 显示区域内的正常交互功能

## 非目标
- 不改变 Imgui 显示区域的大小或形状
- 不修改窗口的整体外观和布局
- 不影响显示区域内的正常交互功能

## 功能需求

### FR-1：限制鼠标捕获范围
- 窗口仅在 Imgui 显示区域内捕获鼠标事件
- 非显示区域（空白/透明区域）应设置适当的窗口样式使鼠标事件穿透

### FR-1.1：显示区域判断
- 系统应能够判断鼠标位置是否在 Imgui 显示区域内
- 显示区域由 Imgui 的绘制边界决定

### FR-1.2：非显示区域鼠标穿透
- 当鼠标位于非显示区域时，窗口不应处理鼠标事件
- 鼠标事件应传递给下层的其他窗口

### FR-1.3：显示区域内正常交互
- 当鼠标位于显示区域内时，窗口应正常处理所有鼠标事件
- 保持原有的拖拽、点击等交互功能

## 技术方案

### 方案分析
1. **窗口样式方案**：通过设置 `WS_EX_TRANSPARENT` 让非显示区域透明并穿透鼠标事件
2. **区域裁剪方案**：使用 `SetCapture` 和区域检测限制鼠标捕获范围
3. **消息过滤方案**：在窗口消息处理中判断鼠标位置，仅处理显示区域内的鼠标消息

### 推荐方案：消息过滤 + 区域检测
- 在窗口的鼠标消息处理中增加位置检测
- 通过 `ImGui::GetCurrentContext()->CurrentWindow->OuterRect` 或类似方法获取 Imgui 显示区域边界
- 仅在显示区域内处理鼠标消息，非显示区域的消息直接透传

## 实现细节

### 需要修改的文件
- `src/main.cpp`：主窗口消息处理
- `include/imgui/backends/imgui_impl_win32.cpp`：Win32 消息处理后端

### 关键实现点
1. 获取 Imgui 显示区域的边界矩形
2. 在鼠标消息处理中添加区域判断
3. 非显示区域的鼠标消息调用 `DefWindowProc` 或返回 `0` 以透传

## 验收标准

### AC-1：非显示区域鼠标穿透
- **给定**：DynamicIsland 窗口已显示在屏幕上
- **当**：用户将鼠标移动到窗口的非显示区域
- **那么**：鼠标事件应传递给下面的窗口，下层窗口应能正常响应鼠标操作
- **验证**：`human-judgment`

### AC-2：显示区域正常捕获
- **给定**：DynamicIsland 窗口已显示在屏幕上
- **当**：用户将鼠标移动到 Imgui 显示区域内
- **那么**：窗口应正常捕获并处理鼠标事件
- **验证**：`human-judgment`

### AC-3：显示区域内交互功能
- **给定**：鼠标在 Imgui 显示区域内
- **当**：用户进行点击、拖拽等操作
- **那么**：窗口应正常响应这些交互
- **验证**：`human-judgment`

## 依赖
- ImGui 库（已集成）
- Win32 API

## 风险与注意事项
- 需要确保不影响现有的文件拖放功能
- 快捷键功能可能需要单独处理
- 窗口边界检测需要考虑多显示器场景
