# DynamicIsland - 限制鼠标捕获范围至Imgui显示区域实现计划

## 任务 1：分析现有鼠标消息处理逻辑
- **优先级**：P0
- **依赖**：无
- **描述**：
  - 分析 `src/main.cpp` 中的窗口消息处理逻辑
  - 分析 `include/imgui/backends/imgui_impl_win32.cpp` 中的鼠标消息处理
  - 找出鼠标消息的处理入口和流程
- **验收标准**：理解鼠标消息的处理流程

## 任务 2：获取 Imgui 显示区域边界
- **优先级**：P0
- **依赖**：任务 1
- **描述**：
  - 研究如何获取 Imgui 当前窗口的显示区域边界
  - 确定使用 `ImGui::GetCurrentContext()` 和相关 API
  - 编写测试代码验证区域获取的正确性
- **验收标准**：能够正确获取 Imgui 显示区域的边界矩形

## 任务 3：实现鼠标位置区域判断函数
- **优先级**：P0
- **依赖**：任务 2
- **描述**：
  - 创建 `IsPointInImguiArea(int x, int y)` 函数
  - 函数判断给定的屏幕坐标是否在 Imgui 显示区域内
  - 处理边界情况（如 Imgui 未初始化）
- **验收标准**：区域判断函数能够正确判断任意点是否在显示区域内

## 任务 4：修改鼠标消息处理逻辑
- **优先级**：P0
- **依赖**：任务 3
- **描述**：
  - 在鼠标消息处理中添加区域判断
  - 非显示区域的鼠标消息调用 `DefWindowProc` 透传
  - 显示区域内的鼠标消息正常处理
- **验收标准**：
  - 非显示区域的鼠标事件不再被捕获
  - 显示区域内的鼠标交互功能正常

## 任务 5：测试验证

  - AC-1：非显示区域鼠标穿透
