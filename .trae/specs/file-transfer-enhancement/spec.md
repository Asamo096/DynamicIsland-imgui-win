# DynamicIsland - 文件中转站功能完善规格文档

## 概述
- **摘要**：完善 DynamicIsland 的文件中转站功能，参考其他厂商（如 macOS 的 AirDrop、Windows 的剪贴板历史）的设计，添加缺失的功能并提升用户体验。
- **目的**：让文件中转站功能更加完整和易用，成为用户日常文件管理的便捷工具。
- **目标用户**：需要频繁处理文件的 Windows 用户，特别是需要在不同应用间快速传递文件的用户。

## 现状分析
- **已实现**：文件拖放、基本文件管理（添加、删除、打开）、简单界面显示
- **缺失**：文件复制/移动功能界面、文件拖出功能、完整的状态显示、详细的文件信息、文件预览

## 功能完善

### FR-1：完善文件管理操作界面
- 添加 "Copy" 和 "Move" 按钮到文件列表
- 实现复制/移动文件的目标路径选择
- 支持批量文件操作

### FR-2：实现文件拖出功能
- 支持将文件从中转站拖放到其他应用
- 处理 Windows 拖放协议
- 确保拖出操作的流畅性

### FR-3：增强状态显示
- 在灵动岛收起状态显示文件数量和总大小
- 在展开状态显示更详细的中转站统计信息
- 添加文件数量限制和存储容量警告

### FR-4：丰富文件信息显示
- 显示文件大小（格式化显示，如 1.5 MB）
- 显示文件添加时间
- 显示文件类型图标或标识
- 支持文件排序（按名称、大小、时间）

### FR-5：添加文件预览功能
- 支持常见文件类型的预览（图片、文本）
- 提供文件详细信息查看
- 增强用户对文件内容的了解

## 技术实现
- **文件操作**：使用 Windows API 进行文件复制/移动
- **拖出功能**：实现 `WM_COPYDATA` 或使用 OLE 拖放
- **界面**：使用 ImGui 实现更丰富的文件信息显示
- **状态管理**：完善 `TransferStation` 类的统计功能

## 验收标准
- AC-1：文件复制/移动功能正常工作
- AC-2：文件拖出功能支持拖放到其他应用
- AC-3：状态显示包含文件数量和总大小
- AC-4：文件信息显示完整（名称、大小、时间）
- AC-5：文件预览功能正常（至少支持图片和文本）

## 参考设计
- **macOS AirDrop**：简洁的文件传输界面
- **Windows 剪贴板历史**：方便的文件管理和预览
- **Microsoft OneDrive**：文件信息的详细显示
