# DynamicIsland - 文件中转站功能完善任务列表

## [ ] 任务 1：完善文件管理操作界面
- **优先级**：P0
- **依赖**：None
- **描述**：
  - 在文件列表中添加 "Copy" 和 "Move" 按钮
  - 实现复制/移动文件的目标路径选择对话框
  - 支持批量文件操作
- **验收标准**：AC-1
- **测试要求**：
  - `programmatic` TR-1.1：验证复制文件功能是否正常
  - `programmatic` TR-1.2：验证移动文件功能是否正常
  - `human-judgment` TR-1.3：验证复制/移动界面是否直观
- **注意**：使用 Windows 通用文件对话框选择目标路径

## [ ] 任务 2：实现文件拖出功能
- **优先级**：P0
- **依赖**：None
- **描述**：
  - 实现文件从中转站拖出的功能
  - 处理 Windows 拖放协议
  - 支持将文件拖放到其他应用
- **验收标准**：AC-2
- **测试要求**：
  - `human-judgment` TR-2.1：验证文件能否成功从中转站拖出
  - `human-judgment` TR-2.2：验证文件能否成功拖放到其他应用
- **注意**：需要实现 `WM_COPYDATA` 或 OLE 拖放机制

## [ ] 任务 3：增强状态显示
- **优先级**：P1
- **依赖**：None
- **描述**：
  - 在灵动岛收起状态显示文件数量和总大小
  - 在展开状态显示更详细的中转站统计信息
  - 添加文件数量限制和存储容量警告
- **验收标准**：AC-3
- **测试要求**：
  - `human-judgment` TR-3.1：验证状态显示是否清晰
  - `programmatic` TR-3.2：验证文件数量和大小统计是否准确
- **注意**：确保状态信息不影响系统监控信息的显示

## [ ] 任务 4：丰富文件信息显示
- **优先级**：P1
- **依赖**：None
- **描述**：
  - 显示文件大小（格式化显示，如 1.5 MB）
  - 显示文件添加时间
  - 显示文件类型图标或标识
  - 支持文件排序（按名称、大小、时间）
- **验收标准**：AC-4
- **测试要求**：
  - `human-judgment` TR-4.1：验证文件信息显示是否完整
  - `human-judgment` TR-4.2：验证文件排序功能是否正常
- **注意**：确保文件信息显示不占用过多界面空间

## [ ] 任务 5：添加文件预览功能
- **优先级**：P2
- **依赖**：任务 4
- **描述**：
  - 支持常见文件类型的预览（图片、文本）
  - 提供文件详细信息查看
  - 增强用户对文件内容的了解
- **验收标准**：AC-5
- **测试要求**：
  - `human-judgment` TR-5.1：验证图片文件预览是否正常
  - `human-judgment` TR-5.2：验证文本文件预览是否正常
- **注意**：预览功能应轻量，不影响性能

## [ ] 任务 6：测试和优化
- **优先级**：P2
- **依赖**：所有任务
- **描述**：
  - 测试不同类型和大小的文件
  - 优化拖放操作的性能
  - 修复可能的 bug 和边缘情况
- **验收标准**：整体功能完整性
- **测试要求**：
  - `programmatic` TR-6.1：测试大文件的操作性能
  - `human-judgment` TR-6.2：验证整体用户体验
- **注意**：关注操作的响应速度和稳定性
